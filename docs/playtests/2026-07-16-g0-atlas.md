## Verification defects found and fixed

The browser pass found four issues that now have red/green regressions:

1. Server/client first render could disagree when local storage already contained a
   Galaxy run. Browser persistence now loads only after hydration.
2. An entry click could theoretically run before the hydration effect. Both experience
   actions remain disabled and guarded until save hydration completes.
3. Galaxy Region POIs did not carry presentation metadata, exposing a faint
   `SECTOR 1-1` compatibility label. Launch and retry now derive the canonical region
   node identity for every POI engine.
4. A truly fresh in-memory save omitted the optional `introSeen` own key, so the strict
   Galaxy projection boundary rejected Ashfall as malformed until reload. Fresh saves
   now own the complete SaveData shape.

The pause and locked-outcome return copy was also aligned with the actual Atlas route.
